Project Data

This project utilized several forms of data, which are described below.

Oracle Data

We used several self-created oracle apps to evaluate M-PERM and P-Lint. Information regarding these apps is available HERE.

Analysis Scripts

The scripts utilized to perform the analysis is available for download from here: https://drive.google.com/uc?id=1xWFeG5KIvZkaCmdGXPB6Y_muTAzudb5E&export=download

Raw Data

The data generated by tools and our analysis have been made available on this website. The dataset is in the format of a single SQLite database file (approximately 6.32 GB).

The (zip compressed) database is available for download from here: https://drive.google.com/uc?id=1y09VBDDAt8K5qpEu9M6gt4lIHBJDj7vd&export=download

Listed below is a brief descriptions of the columns in each table

Table: git_commit_log

Column Name Type Purpose
App Text The name of the app (unique)
Commit_SHA Text The SHA (id) of the commit
Message Text The commit message
Message_Short Text A shortened version of the commit message
Author_Name Text The name of the commit author
Author_Email Text The email of the commit author
Author_Date_Ticks Float The date (in ticks) when the author staged the commit
Author_Date_Text Text The date (in text) when the author staged the commit
Committer_Name Text The name of the person who committed the update
Committer_Email Text The email of the person who committed the update
Committer_Date_Ticks Float The date (in ticks) when the committer pushed the commit
Committer_Date_Text Text The date (in text) when the committer pushed the commit

Table: git_commit_files

Column Name Type Purpose
App Text The name of the app (unique)
Commit_SHA Text The SHA (id) of the commit
Path Text The relative path of the file in the repository
Old_Path Text The relative path of the file in the repository prior to the file being renamed
Status Text The type of action performed on the file (i.e., added, modified, deleted or renamed the file)

Table: git_commit_file_OnlyJavaAndManifestFiles

Column Name Type Purpose
App Text The name of the app (unique)
Commit_SHA Text The SHA (id) of the commit
Path Text The relative path of the file in the repository
Old_Path Text The relative path of the file in the repository prior to the file being renamed
Status Text The type of action performed on the file (i.e., added, modified, deleted or renamed the file)

Table: permission_types

Column Name Type Purpose
Permission_Id Integer The id of the permission (unique)
Permission_Name Text The name of the permission
Permission_Type Text The category the permission belongs to
Is_Dangerous Integer If the permission is known to be a dangerous permission, the value of this column is 1, otherwise 0

Table: mperm_analyzed

Column Name Type Purpose
Index Integer The primary key column of the table
App_Name Text The name of the app
Commit_Id Text The commit SHA (id) containing the permission
Permission_Name Text The name of the permission
Is_Requested Integer 1 if the app requests the permission from the user
Is_Third_Party Integer 1 if permission is 3rd party
Is_Dangerous Integer 1 if the permission is dangerous
Is_Over_Requested Integer 1 if the permission is an over request
Timeunix Float The time of the commit
RQ1 Text This column corresponds to the ‘Is_Requested’ value of mperm. Possible values:
  • New: permission is introduced in this commit
  • Exist:permission existed in previous commit
  • Fix: permission no appears in subsequent commit
RQ2 Text This column corresponds to the ‘is_third_party’ value of mperm and have the possible values of new, exist, fix.
RQ3 Text This column corresponds to the ‘is_dangerous’ value of mperm and have the possible values of new, exist, fix.
RQ4 Text This column corresponds to the ‘is_over_requested’ value of mperm and have the possible values of new, exist, fix.

Table: plint_analyzed

Column Name Type Purpose
App_Name Text The name of the app
Commit_Id Text The commit SHA (id) containing the permission
File_Name Text The path of the file containing the issue
Method_Name Text The name of the method associated with the issue
Author_Email Text The author of the commit
Timeunix Text The timestamp of the commit
UseCase 1 … UseCase 16 Text These columns correspond to the use cases checked by plint. An ‘X’ indicates the presence of an issue, while ‘O’ indicates the absence of an issue.
UC 1 … UC 16 Text These columns provide more details on the status of the permission issue. If the issue was newly added, removed or is an existing issue.

Table: plint_analyzed_dcr

Column Name Type Purpose
Index Integer The primary key column of the table
App_Name Text The name of the app
Commit_Id Text The commit SHA (id) containing the permission
Author_Email Text The name of the author of the update
DCR Text The DCR score of the author performing the commit

Table: mperm_analyzed_source_code_permissions

Column Name Type Purpose
App_Name Text The name of the app
Commit_Id Text The commit SHA (id) containing the permission
Permission_Name Text The name of the permission detected by mperm
File_Path Text The absolute path of the file
Source_Code Text The source code statement containing the permission
Line Integer The line number of the file containing the permission

Table: mperm_analyzed_error_log

Column Name Type Purpose
Index Text The primary key column of the table
App_Name Text The name of the app
APK_ID Text The unique id of the apk file
File_Path Text The absolute path of the Android Manifest file
Error_Message Text The error encountered by the tool when analyzing the file

Table: Cloned_app

Column Name Type Purpose
App Text The name of the app

Table: Cloned_app_with_Java_Manifest

Column Name Type Purpose
App Text The name of the app

Table: Cloned_app_without_Java_Manifest

Column Name Type Purpose
App Text The name of the app
